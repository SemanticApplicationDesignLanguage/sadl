How to Make a New Release

Last revised 09/14/2020.

To make a new SADL release, do the following.

  1. First change the version number unless only the build is to be updated. See Update Sadl 3 Version.
  2. Go to the github repository and click on "Releases" (currently on the right of the list of changes).
  3. Click on "Draft a new release". You may wish to have the releases list open in another browser window to compare prior release content while creating a new release.
  4. Fill in the content of the new release
    • The "Tag version" should normally be of the form <version-number>.<build>, e.g., 3.4.1.202009041213.
    • Fill in title and description as desired.
    • Attach the binary zip file, found in the project com.ge.research.sadl.update, target folder, e.g., .../com.ge.research.sadl.update/target/com.ge.research.sadl.update-3.4.1.202009141800.zip
    • If this is a pre-release, check the box.
  5. Click on "Publish release". Note that the release can be edited if need be.
