Update Version Number in SADL 3

Last revised 09/04/2020.

To update the version number, e.g., from SADL 3.3.0-SNAPSHOT to SADL 3.4.0-SNAPSHOT, do the following:

  • update the <version> and the <sadlOsgiVersion> in the com.ge.research.sadl.parent project
  • update the <version> within the <parent> tag for each projects's pom.xml file
    •
  • update the Bundle-Version: in the MANIFEST.MF file for each project that has one
    • Note that a few projects do not have MANIFEST.MF files.
    • Don't forget to drill down into nested projects such as com.ge.research.sadl.reasoner
  • update the version in feature.xml in the SADL feature project com.ge.research.sadl.feature.

It's not a bad idea to do a search for the old version number in .xml and .MF files to make sure that nothing has been missed. (Note that references under the target directory won't be updated until a clean/build is performed.)

Note, updating the category.xml for the p2 is not required as it is set to 0.0.0.

For a reference commit in github, see https://github.com/crapo/sadlos2/pull/419/commits/1416537ae063902d1f519caaa86350c2b70ac754.
