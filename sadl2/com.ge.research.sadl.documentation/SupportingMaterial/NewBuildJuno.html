Instructions for Creating SADL Development Environment for Eclipse Juno, Xtext 2.3.1, SADL 2.2

Starting with SADL 2.2.0 for Eclipse Juno (with Xtext 2.3), the entire build process has been Mavenized so that Eclipse update sites can be built using Maven. This will greatly decrease the amount of time required to build an update.

Downloads and Installs

  1. Download the complete Eclipse Juno with Xtext from http://eclipse.org/Xtext.
  2. Make sure that Apache Maven 3.0 is installed on your machine, available from http://maven.apache.org/, then follow the installation instructions at the bottom of the download page.
  3. Configure Maven to use the desired repositories with modifications to the .../.m2/settings.xml file
  4. In the Eclipse environment, check out projects from SourceForge, other repositories as needed.(Or do directly with other source code control client.)
  5. Open command-line window and change directory to the com.ge.research.sadl.build folder.
  6. Run mvn clean install (Options include "-U" to force refresh from Nexus server, -X for complete debug output).
  7. Result should be a complete build of the update site. Install in fresh Eclipse Juno and test.
  8. Note: refreshing the projects in Eclipse should also create a valid development environment. Run the com.ge.research.sadl.ui project as an Eclipse Application. Note that the workspace must be set to one without spaces in the path, e.g., "${workspace_loc}/../runtime-EclipseApplication" and the JVM args may need setting to provide more memory, e.g., "-Xms120m -Xmx1024m".
